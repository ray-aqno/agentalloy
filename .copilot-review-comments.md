## 1. Redundant auto-discovery imports
**File**: `src/agentalloy/providers/__init__.py:44`
**Severity**: Medium

Provider auto-discovery imports each provider module 5 times (lines 35-44). While Python caches imports, the repeated calls are unnecessary overhead and can cause confusing side effects if a module relies on import-time behavior beyond simple registration.

---

## 2. Child process process group safety
**File**: `src/agentalloy/install/subcommands/wrap.py:256`
**Severity**: High

The child process is started without a new session, but the signal handler later calls `os.killpg(os.getpgid(proc.pid), ...)`. If the child shares the same process group as wrap, this can signal the entire group (including the wrapper itself and potentially the user's shell). Start the child in a new session/process group to make killpg safe.

## 4. tool_name always None in hook router
**File**: `src/agentalloy/api/hook_router.py:128`
**Severity**: Medium

_build_predicate_context is being passed `tool_name=getattr(Request, "tool_name", None)`, which reads an attribute off FastAPI's Request class (not the incoming payload) and will always be None. This should be a real tool name value or explicitly None.

## 7. Duplicate registry key — github-copilot
**File**: `src/agentalloy/providers/copilot_cli/__init__.py:54`
**Severity**: High

This provider registers REGISTRY["github-copilot"], but src/agentalloy/providers/github_copilot/__init__.py also registers the same key. With auto-discovery, one will overwrite the other depending on import order, which makes the registry nondeterministic. Consider consolidating into a single provider package or using distinct registry keys.

## 9. Legacy wire_harness KeyError
**File**: `src/agentalloy/install/subcommands/wire_harness.py:478`
**Severity**: Medium

_wire_legacy() assumes the harness exists in the legacy _HARNESS_REGISTRY (reg = _HARNESS_REGISTRY[harness]). However, wire_harness() now validates harnesses against agentalloy.providers.REGISTRY, so a harness can pass validation but still crash here with KeyError when legacy=True. It's better to guard and raise a user-facing SystemExit with a clear message.
